# Guide de contribution — Space League

## Nature du dépôt

Ce dépôt est un **projet propriétaire et vitrine**. Les contributions externes ne sont pas ouvertes
par défaut comme sur un projet open source classique.

## Ce qui est utile

- **Signalement de bugs ou incohérences** dans le code ou la documentation
- **Retours techniques** sur l'architecture, la lisibilité ou les patterns utilisés
- **Suggestions** sur la documentation ou la présentation du projet

## Comment contribuer

### 1. Ouvrir une issue

Avant tout, ouvrez une issue pour décrire le besoin :
- Titre clair et précis
- Contexte et justification
- Impact estimé et périmètre concerné
- Attendre la validation de l'équipe avant d'engager un changement

### 2. Processus de review

Toute proposition sera évaluée en fonction de :
- Sa pertinence par rapport aux objectifs du projet
- Sa cohérence avec l'architecture existante
- La qualité et la lisibilité du code proposé

### 3. Standards de code

- **C++** : suivre les conventions UE5 (préfixes A/U/F, PascalCase)
- **Blueprints** : nommage explicite, commentaires sur les nœuds complexes
- **Documentation** : mettre à jour les specs si une mécanique change
